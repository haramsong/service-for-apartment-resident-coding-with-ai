# 회원가입 아파트 선택 UI/UX 개선

*작성일: 2025-11-03*

*작성자: Developer Agent*

*이슈 유형: UI/UX 개선*

*우선순위: Medium*

## 📋 문제 상황

### 기존 문제점
- 아파트 ID를 직접 텍스트로 입력해야 함
- 사용자가 아파트 ID를 알 수 없음
- 잘못된 ID 입력 시 에러 발생
- 동/호수 입력 가이드 부족

## 🔍 원인 분석

### 1. 사용자 경험 문제
- 아파트 ID는 시스템 내부 식별자로 사용자가 알 수 없는 정보
- 텍스트 입력 방식은 오타 발생 가능성 높음
- 입력 가이드 부족으로 혼란 발생

### 2. UI 구조 문제
- 모든 입력 필드가 동일한 중요도로 표시
- 아파트 선택과 동/호수 입력의 관계가 명확하지 않음
- 시각적 피드백 부족

## ✅ 해결 방법

### 1. Select 컴포넌트 도입
```typescript
// shadcn/ui Select 컴포넌트 추가
bunx shadcn@latest add select
```

### 2. 아파트 선택 UI 개선
- **드롭다운 방식**: 아파트 목록에서 선택
- **상세 정보 표시**: 아파트명 + 주소 표시
- **선택 확인**: 선택한 아파트 정보를 카드로 표시

### 3. 단계별 입력 플로우
1. 기본 정보 입력 (이메일, 비밀번호, 이름)
2. 아파트 선택
3. 동/호수 입력 (아파트 선택 후에만 표시)

### 4. 시각적 개선
- **아이콘 추가**: Building2, MapPin 아이콘으로 직관성 향상
- **섹션 구분**: border-t로 정보 그룹 구분
- **선택 피드백**: 선택한 아파트를 파란색 카드로 표시
- **입력 가이드**: placeholder와 설명 텍스트 추가

## 📊 구현 결과

### 개선된 기능
- ✅ 아파트 드롭다운 선택
- ✅ 아파트명 + 주소 표시
- ✅ 선택 확인 카드
- ✅ 조건부 동/호수 입력 (아파트 선택 후)
- ✅ 입력 가이드 및 예시
- ✅ 에러 메시지 개선

### 사용자 플로우
1. 이메일, 비밀번호, 이름 입력
2. 아파트 드롭다운에서 선택
3. 선택한 아파트 정보 확인
4. 동/호수 입력 필드 표시
5. 동/호수 입력 (예: 101동, 1001호)
6. 회원가입 완료

## 🔄 예방 방법

### 1. 아파트 데이터 관리
- 실제 환경에서는 API로 아파트 목록 조회
- 검색 기능 추가 고려 (아파트가 많을 경우)

### 2. 입력 검증 강화
- 동/호수 형식 검증 추가
- 중복 가입 방지
- 아파트 선택 필수 검증

### 3. 사용자 가이드
- 툴팁 추가
- 도움말 링크
- 예시 이미지
